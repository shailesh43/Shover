Whether you're fixing a bug, improving the documentation, designing a new component, or suggesting an idea, every contribution helps make Shover better for everyone. Shover is built in public, and community contributions are a huge part of its growth.

## Getting Started

1. Fork the repository.
2. Clone your fork locally.
3. Create a new branch for your changes.
4. Make your changes and test them thoroughly.
5. Commit using a clear, descriptive message.
6. Push your branch and open a Pull Request.

```bash title=".sh"
git clone https://github.com/shailesh43/Shover.git
cd Shover
git checkout -b feature/new-feat
```

## Before Opening a Pull Request

Please make sure that:

* Your code follows the existing project structure and style.
* Components are responsive and accessible.
* New features include documentation when applicable.
* Existing functionality isn't unintentionally affected.


## Code of Conduct

Please be respectful, constructive, and welcoming to everyone in the community. We want Shover to be a place where developers feel comfortable learning, collaborating, and sharing ideas.

## Need Help?

If you have questions, ideas, or aren't sure where to start, feel free to open a **GitHub Discussion** or create an **Issue**.
